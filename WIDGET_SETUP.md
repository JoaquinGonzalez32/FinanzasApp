# Widget Setup Instructions

## Architecture Overview

```
React Native App
    |
    v
useWidgetSync() hook (debounced, event-driven)
    |
    v
buildWidgetData() ── fetches from Supabase, builds WidgetData
    |
    v
widgetBridge.ts ── NativeModules.RNWidgetBridge
    |
    +──> Android: SharedPreferences + AppWidgetManager
    +──> iOS: App Groups UserDefaults + WidgetCenter
```

**Data flow**: App changes -> event emitted -> useWidgetSync fires -> buildWidgetData fetches fresh data from Supabase -> bridge writes to native storage -> native widgets re-render.

**Deep links**: Widget tap -> `finanzasapp://add?category=X&account=Y` -> RootNavigator handles -> routes to correct screen with params.

## iOS Setup (Manual Steps Required)

Since `ios/` is generated by EAS Build, the widget extension needs to be added via Xcode after the first build.

### Step 1: Generate the iOS project
```bash
npx expo prebuild --platform ios
```

### Step 2: Copy bridge files into the iOS project
```bash
cp ios-widget/RNWidgetBridge.swift ios/FinanzaApp/
cp ios-widget/RNWidgetBridge.m ios/FinanzaApp/
```

### Step 3: Add Widget Extension target in Xcode

1. Open `ios/FinanzaApp.xcworkspace` in Xcode
2. File > New > Target > Widget Extension
3. Name it `FinanzaWidgetExtension`
4. Uncheck "Include Configuration App Intent"
5. Replace the generated files with the contents from `ios-widget/FinanzaWidgetExtension/`:
   - `FinanzaWidget.swift`
   - `WidgetDataProvider.swift`
   - `WidgetDataReader.swift`
   - `Views/SummaryWidgetView.swift`
   - `Views/QuickAddWidgetView.swift`
   - `Views/LastTransactionView.swift`

### Step 4: Configure App Groups

1. Select the **main app target** > Signing & Capabilities > + Capability > App Groups
2. Add group: `group.com.joacobaffe.finanzasapp`
3. Select the **widget extension target** > same steps
4. Add the same group: `group.com.joacobaffe.finanzasapp`

### Step 5: Add bridge files to main target

1. In Xcode, drag `RNWidgetBridge.swift` and `RNWidgetBridge.m` into the main app target
2. When prompted for bridging header, accept (or ensure your existing one imports React)
3. Add `WidgetKit` framework to the main app target (linked, not embedded)

### Step 6: Configure EAS Build

Add to `app.json` under `ios`:
```json
{
  "ios": {
    "appGroupIdentifiers": ["group.com.joacobaffe.finanzasapp"]
  }
}
```

Or use an Expo config plugin to automate the widget extension setup.

### Step 7: Build & Test
```bash
eas build --platform ios --profile development
```

## React Native Layer

### Files Created
- `src/features/widgets/types/widget.types.ts` — WidgetData, WidgetConfig types
- `src/features/widgets/services/widgetData.ts` — Builds WidgetData from Supabase
- `src/features/widgets/services/widgetConfig.ts` — Persists user widget preferences
- `src/features/widgets/services/widgetBridge.ts` — JS wrapper for native module
- `src/features/widgets/hooks/useWidgetSync.ts` — Auto-sync on data changes
- `src/features/widgets/hooks/useWidgetConfig.ts` — Hook for widget settings UI
- `app/widget-settings.js` — Widget configuration screen

### Files Modified
- `app/_layout.js` — Added useWidgetSync, deep link handler, widget-settings route
- `app/add-transaction.js` — Accept `category` and `account` params from deep links
- `app/(tabs)/settings.js` — Added "Widgets" tool entry

### How it works
1. `useWidgetSync()` runs in `RootNavigator` — listens to transaction/budget/account events
2. On any change (debounced 2s), it calls `buildWidgetData()` which fetches fresh data
3. Data is serialized to JSON and written to native storage via the bridge
4. Native widgets read from that storage and render

## Deep Link Scheme

| Widget Action | Deep Link | Target Screen |
|---|---|---|
| Tap category button | `finanzasapp://add?category={id}&account={id}` | Add Transaction (preselected) |
| Tap summary | `finanzasapp://budget` | Budget tab |
| Tap last transaction | `finanzasapp://transactions` | Transactions tab |
| Tap header/logo | `finanzasapp://home` | Home tab |

The scheme `finanzasapp` is already configured in `app.json` and handled in `_layout.js`.
